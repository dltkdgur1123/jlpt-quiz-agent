# NS-TICKET-006: 정답 제출 API

## 역할

사용자가 선택한 답안을 저장하고 정답/오답 결과를 돌려주는 단계다.

## 왜 이렇게 구축했는가

풀이 기록은 반복 제출이 가능해야 하지만, 출제 경험 feedback과는 분리되어야 한다. 그래서 정답 제출 API는 `quiz_attempts` 저장과 정답 확인만 담당하고, feedback은 이후 별도 API에서 처리하도록 나눴다.

## AI 에이전트 역할

Hermes Chief Agent가 QA Agent 관점으로 먼저 실패 테스트를 만들고, Coding Agent 관점으로 정답 계산과 저장 흐름을 구현했다.

## 결과

정답/오답 계산, 반복 풀이 허용, feedback 미저장 기준이 테스트로 고정되었다.
