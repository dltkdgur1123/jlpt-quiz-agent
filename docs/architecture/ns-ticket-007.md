# NS-TICKET-007: 출제 경험 feedback API

## 역할

정답 확인 이후 로그인 사용자가 “시험에서 본 기억”을 제출하거나 수정할 수 있게 하는 단계다.

## 왜 이렇게 구축했는가

출제 체감 score는 사용자 제보를 기반으로 하므로 비로그인 제출은 막고, 같은 사용자가 같은 문항에 여러 번 누적하지 않도록 update 방식으로 처리한다.

## AI 에이전트 역할

Hermes Chief Agent가 Trust/QA 관점으로 로그인 제한과 중복 누적 방지 테스트를 먼저 만들고, Coding Agent 관점으로 최소 API 흐름을 구현했다.

## 결과

feedback 값은 `yes / no / unknown`만 허용되고, 비로그인 제출은 차단되며, 기존 응답은 업데이트되는 기준이 생겼다.
