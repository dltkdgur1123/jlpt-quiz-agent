# 오답노트 복습 허브 설계 v1

## 결정

오답노트는 단순한 `틀린 문제 목록`이 아니라, 모의고사 제출 이후 사용자가 다시 학습을 시작하는 **복습 허브**로 설계한다.

```text
모의고사 결과
→ 오답/미응답 저장
→ 학습기록의 오답노트
→ 다시 풀기 / 해설 확인 / 복습 완료
→ 취약 영역 분석 갱신
```

## 역할

1. **오답·미응답 수집**
   - 모의고사 제출 시 틀린 문제와 풀지 않은 문제를 저장한다.
   - 결과 화면 CTA의 목적지가 된다.

2. **복습 실행 시작점**
   - 학습기록 페이지에서 사용자가 바로 다음 행동을 할 수 있게 한다.
   - 핵심 행동은 `오답노트 보기`, 이후 `다시 풀기`, `해설 보기`, `복습 완료`로 확장한다.

3. **취약 영역 분석의 원천**
   - 오답노트에 쌓인 문제를 영역별로 집계한다.
   - 문자·어휘 / 문법 / 읽기 단위로 취약 영역 분석에 연결한다.

4. **학습기록 페이지의 행동 카드**
   - 최근 모의고사: 과거 기록 확인
   - 오답노트: 지금 복습 실행
   - 취약 영역 분석: 앞으로의 학습 방향 판단

## Dashboard 배치

권장 배치:

```text
[학습 요약 4개]

[주간 학습 활동]        [이번 달 목표]
[최근 모의고사]        [오답노트]

[취약 영역 분석 - 전체 폭]
```

- `오답노트`는 오른쪽 하단 카드에 둔다.
- `취약 영역 분석`은 별도 전체 폭 섹션으로 둔다.
- 두 섹션은 경쟁 관계가 아니라 연결 관계다.

## MVP 표시 범위

학습기록 페이지의 오답노트 카드는 처음에는 compact summary로 시작한다.

```text
오답노트
틀렸거나 풀지 않은 문제를 다시 확인하세요.

오답 n문항
미응답 n문항

최근 복습 대상
- 문자·어휘 n문항
- 문법 n문항
- 읽기 n문항

[오답노트 보기]
```

## MVP 데이터 기준

- 1차 데이터 소스: `mock_exam_answers`
- 포함 대상:
  - `is_correct = false`
  - 미응답/blank answer
- 영역 라벨:
  - `vocab` → `문자·어휘`
  - `grammar` → `문법`
  - `reading` → `읽기`
- 청해 제외 세트에서는 `듣기`를 표시하지 않는다.

## 확장 범위

나중에 별도 오답노트 페이지 또는 확장 패널에서 추가한다.

1. 다시 풀기 큐
   - 오답만 다시 풀기
   - 미응답만 다시 풀기
   - 영역별 다시 풀기

2. 복습 상태
   - 미복습
   - 다시 틀림
   - 복습 완료

3. 반복 오답
   - 2회 이상 틀린 문제 강조
   - 마지막 복습일 표시

4. 해설 중심 학습
   - 정답
   - 내가 고른 답
   - 해설
   - 왜 틀렸는지 개인 메모

5. 취약 영역 반영
   - 오답노트의 영역별 집계가 `취약 영역 분석`에 반영된다.

## 금지/주의

- 결과 화면에 긴 오답 리스트를 다시 넣지 않는다.
- 오답노트를 단순 기록 저장소처럼만 보이게 하지 않는다.
- 공식 합격/불합격 예측처럼 보이는 문구와 연결하지 않는다.
- 실제 데이터가 없을 때 fake 오답/취약 영역을 보여주지 않는다.
- 로그인 전에는 저장 이점을 안내하고, 실제 기록처럼 보이는 숫자는 피한다.
